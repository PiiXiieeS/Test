Classes

Introduction

In this article you can find three ways to define an object. It is important to note that there are no classes in JavaScript but functions can be used to somewhat simulate classes, but in general JavaScript is a class-less language.

Using a function

You define a normal JavaScript function and then create an object by using the new keyword. Properties and methods can be added with the keyword this.

function Apple (type) {
    this.type = type;
    this.color = "red";
    this.getInfo = getAppleInfo;
}

// anti-pattern! keep reading...
function getAppleInfo() {
    return this.color + ' ' + this.type + ' apple';
}

// and objects are instantiated using the constructor function
var apple = new Apple('macintosh');
apple.color = "reddish";
alert(apple.getInfo());

In this example the method getInfo() was defined in a separate function getAppleInfo(). While this works fine, you may end up defining a lot of these functions and they are all in the global namespace.

There are two ways to prevent the pollution of the global namespace.

Methods defined internally

You can define your methods within the constructor function.

function Apple (type) {
    this.type = type;
    this.color = "red";
    this.getInfo = function() {
        return this.color + ' ' + this.type + ' apple';
    };
}

The drawback is that the method getInfo() is recreated every time you create a new object.

Methods added to the prototype

A better way is to add getInfo() to the prototype of the constructor function.

function Apple (type) {
    this.type = type;
    this.color = "red";
}

Apple.prototype.getInfo = function() {
    return this.color + ' ' + this.type + ' apple';
}

Using object literals

Literals are shorter way to define objects and arrays in JavaScript.

To create an empty object using you can do: var o = {}; instead of the "normal" way: var o = new Object();

For arrays you can do: var a = []; instead of: var a = new Array();

So you can directly create an instance of an object using this object literal syntax.

var apple = {
    type: "macintosh",
    color: "red",
    getInfo: function () {
        return this.color + ' ' + this.type + ' apple';
    }
}
//the object is already instantiated
apple.color = "reddish";
alert(apple.getInfo());

Such an object is also sometimes called singleton. As in JavaScript there are no classes, all objects are singletons to begin with.

Singleton using a function

The third way is a combination of the other two methods described above. We use a function to define a singleton object.

var apple = new function() {
    this.type = "macintosh";
    this.color = "red";
    this.getInfo = function () {
        return this.color + ' ' + this.type + ' apple';
    };
}
// and the object is already instantiated
apple.color = "reddish";
alert(apple.getInfo());
